Location and Maps Programming Guide

推迟更新可以让你的app沉睡一段时间从而可以节省资源。因为推迟位置更新需要全球定位系统硬件的存在，所以在这之前需要调用CLLocationManager类方法deferredLocationUpdatesAvailable去检测当前设备是否支持推迟位置更新

调用CLLocationManager类方法allowDeferredLocationUpdatesUntilTraveled:timeout:开始推迟位置更新。

// Delegate method from the CLLocationManagerDelegate protocol.
- (void)locationManager:(CLLocationManager *)manager
     didUpdateLocations:(NSArray *)locations {
    // Add the new locations to the hike
    [self.hike addLocations:locations];
    // Defer updates until the user hikes a certain distance
    // or when a certain amount of time has passed.
    if (!self.deferringUpdates) {
        CLLocationDistance distance = self.hike.goal - self.hike.distance;
        NSTimeInterval time = [self.nextAudible timeIntervalSinceNow];
        [locationManager allowDeferredLocationUpdatesUntilTraveled:distance
         self.deferringUpdates = YES;
    }

}

当符合如上所述的条件时，调用allowDeferredLocationUpdatesUntilTraveled:timeout:后，local manager就会调用locationManager:didFinishDeferredUpdatesWithError:让委托对象知道已经停住并推迟发送位置更新了。每当调用allowDeferredLocationUpdatesUntilTraveled:timeout:时，local manager就会调用这个代理方法。在推迟更新结束之后，local manager发送所有的位置更新给委托方法locationManager:didUpdateLocations:

你可以通过调用disallowDeferredLocationUpdates方法来明确的阻止推迟位置更新，当你调用这个方法，或者通过stopUpdatingLocation方法来停止位置更新服务时，local manager将会调用locationManager:didFinishDeferredUpdatesWithError:来确认推迟更新确实已经被终止了。

如果local manager发生了错误并且不能延迟位置更新时，你可以通过实现locationManager:didFinishDeferredUpdatesWithError:代理方法来获取产生错误的原因。
